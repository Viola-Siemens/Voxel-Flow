# VoxelFlow - A project management platform for Minecraft mods

## 设计前提

本项目预期实现一个简易的项目管理平台 Voxel Flow，旨在帮助 Minecraft 相关产品（模组、插件、建筑等）研发团队制定和监督完成计划，提高效率。

运营模式：
1. 业务方在平台创建新的 Requirement，审批后产品经理需要对需求进行分析，产出产品需求文档（PRD）并进行评审；
2. PRD 评审通过后，研发工程师则根据 PRD 产出技术设计文档，技术设计评审通过后启动开发，测试工程师则同时根据 PRD 产出测试用例；
3. 研发工程师在完成开发后提测，测试工程师根据测试用例完成测试，测试通过后提交业务方验收；
4. 验收成功则发布，完成整个产研流程。

文档撰写在外部的文档平台完成，而代码提交和版本控制则在 GitHub 实现，并通过 Webhook 回调通知本平台。本平台最终仅仅实现项目管理和产研流程记录的基本功能。
